# 锐历 Beta 验收测试清单 (Acceptance Checklist)

## 1. 注册登录
- **测试步骤**：新用户在首页点击注册，输入邮箱密码进行注册；模拟断开或未配置 SMTP 的环境测试兜底逻辑。
- **预期结果**：能正常注册并成功跳转到 Dashboard；无 SMTP 也能通过系统默认处理或日志确认注册；用户凭证正确写入 DB。
- **失败时影响**：新用户完全无法进入产品，核心转化归零。
- **是否阻塞上线**：是 (Blocker)

## 2. 简历创建
- **测试步骤**：在 Dashboard 点击“新建简历”，填写基础信息进入编辑器，添加至少一段工作经历和教育经历。
- **预期结果**：表单数据实时保存，并无缝渲染同步到右侧预览区，无报错或白屏。
- **失败时影响**：核心链路中断，无法进行后续编辑操作。
- **是否阻塞上线**：是 (Blocker)

## 3. 模板切换
- **测试步骤**：在编辑器左侧配置面板点击“模板”，在多款自带模板（如单栏、双栏）间快速点击切换。
- **预期结果**：右侧简历预览实时更新排版样式，表单数据不丢失，页面不发生崩溃或闪退。
- **失败时影响**：用户无法选择适配自身经历的简历样式，产品体验大幅降级。
- **是否阻塞上线**：是 (Blocker)

## 4. PDF 导出
- **测试步骤**：在编辑器内点击右上角“导出” -> “导出为 PDF”，分别针对一页纸内容和超长多页纸内容进行测试。
- **预期结果**：前端触发并成功下载 PDF 文件，排版与编辑器右侧预览高度一致，文本可选中且中文不乱码。
- **失败时影响**：用户无法获取最终求职产物，产品完全失去工具属性。
- **是否阻塞上线**：是 (Blocker)

## 5. DOCX 导出
- **测试步骤**：在编辑器内点击“导出” -> “导出为 DOCX”，使用 Microsoft Word 或 WPS 打开生成的文件。
- **预期结果**：成功下载 DOCX，基础文字内容完整。对于复杂排版模板，允许降级为流式文本排版。
- **失败时影响**：失去喜欢在 Word 中二次调格式的用户群体，但不影响主推的 PDF 交付工作流。
- **是否阻塞上线**：否 (Nice to have，允许一定程度的样式降级)

## 6. PDF/DOCX/图片导入
- **测试步骤**：在系统设置配置好 AI Provider 与 OCR Provider 后，在 Dashboard 上传一份标准 PDF 和一张 JPG 简历截图。
- **预期结果**：后台解析成功，在编辑器中还原出相应的字段结构，能够正确提取出姓名、联系方式、工作经历等核心数据。
- **失败时影响**：用户必须从零手写填表，难以吸引手中已有旧简历的用户迁移至锐历。
- **是否阻塞上线**：是 (Blocker，这是 AI 赋能的重点卖点)

## 7. AI Provider 未配置
- **测试步骤**：使用刚注册的新账号（未配置任何 API Key），直接上传 PDF 或 DOCX 文件要求导入。
- **预期结果**：导入动作被明确拦截，UI 给出清晰具体的报错信息（如“需要配置 AI Provider”），并提供快捷跳转至设置页的入口。
- **失败时影响**：报错抛出晦涩的网络错误或服务端状态一直 Pending，用户不知所措。
- **是否阻塞上线**：是 (Blocker，异常处理必须完备)

## 8. OCR Provider 未配置
- **测试步骤**：清除浏览器所有缓存，使用新账号直接上传 JPG / PNG 图片简历。
- **预期结果**：前端立刻拦截并提示缺少 OCR Provider 设置，引导用户配置，防止白白向 AI Provider 发起无效调用或导致解析失败。
- **失败时影响**：图片解析完全崩溃或乱码，并且不给出具体指引，导致体验中断。
- **是否阻塞上线**：是 (Blocker，交互防呆链路必须闭环)

## 9. 1核1G VPS 部署风险
- **测试步骤**：在 1C1G 规格的单机节点上按最低标准部署服务，且未配置 Swap，发起 5-10 个并发的 PDF 导出请求。
- **预期结果**：Puppeteer 服务可能变慢甚至引发 OOM 崩溃，但 Docker 的 Restart 策略能够拉起进程恢复服务；且文档（如 `chinese-resume-beta.md`）必须有高亮警告，要求小内存用户开启 Swap。
- **失败时影响**：自建站长或开源使用者极易宕机，败坏项目口碑。
- **是否阻塞上线**：是 (Blocker，文档警示必须到位)

## 10. 对比 OrtonY/smart-resume 后的 Beta 决策

根据与核心竞品 `OrtonY/smart-resume` 的详细对比分析，特界定以下 Beta 版取舍边界：

- **Beta 必须做 (Blockers)**：
  - 流畅的外部文件导入与高准确率结构化解析（不能让用户重抄一遍）。
  - 可靠的 PDF 导出与无损的中文排版支持（核心交付物）。
  - 清晰的错误拦截提示与 AI 配置引导（降本增效、降低理解成本）。
  - 稳定闭环的自托管部署方案指南（面向开源生态圈）。

- **Beta 可以延后 (Nice to have)**：
  - 编辑器内“单条工作经历的 AI 深度改写”。（初期优先打透“导入”场景，后期再迭代编辑器智能）
  - 针对目标 JD 自动化派生专属简历版本。（初期可让用户手动“复制”副本自行微调）
  - 更多国内 OCR 厂商生态集成。（初期可先用 Azure 方案顶住，后续再接入阿里/百度，甚至直接拥抱多模态 LLM 免 OCR 直读）
  - 完美的高保真 DOCX 导出引擎。（初期提供基础的文字流式保底即可，力推 PDF）
